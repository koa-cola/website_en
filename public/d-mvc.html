MVC pattern with decorator

In koa-cola we can write mvc by using es7's decorator. Controller have to be defined with the provided decorator (as it relates to the router related definition), and the model and view layers are not forced to be defined by the decorator as the demo following.

Custom header & bundle packing

When koa-cola render view in server side, it will be looking for `views/pages/layout.ts` as the page layout. if `layout.ts` file does not exist, the view component will render directly. If the view component uses the `doNotUseLayout` decorator, the page will not use `layout.ts`, and you probably need `header` and `bundle` decorators to define header and resource.
